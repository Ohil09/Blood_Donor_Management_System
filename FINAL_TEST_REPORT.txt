📋 CASCADING CLEANUP PROCESS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a hospital is deleted, the following operations occur automatically:

Step 1: Unassign Donors
   └─ db.users.update_many() → hospital_id = NULL
   └─ Effect: 2 donors unassigned

Step 2: Remove Admin Accounts
   └─ db.users.delete_many() → hospital_id matches
   └─ Effect: 1 admin user deleted

Step 3: Cancel Donation Requests
   └─ db.donation_requests.update_many() → status = "cancelled"
   └─ Effect: 2 requests cancelled + audit trail

Step 4: Cancel Exchange Requests
   └─ db.inter_hospital_requests.update_many() → status = "cancelled"
   └─ Effect: 2 requests cancelled + audit trail

Step 5: Remove Inventory
   └─ db.inventory.delete_many() → hospital_id matches
   └─ Effect: Inventory deleted

Step 6: Delete Hospital
   └─ db.hospitals.delete_one() → hospital_id matches
   └─ Effect: Hospital removed from system

Step 7: User Feedback
   └─ Flash success message
   └─ Redirect to dashboard
